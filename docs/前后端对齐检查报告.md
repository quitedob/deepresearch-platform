## 一、用户认证模块对齐情况

### 1.1 前端 API 调用
**位置**: `vue/src/api/user.js` 和 `vue/src/api/index.js`

前端调用的接口：
- `POST /api/v1/auth/register` - 用户注册
- `POST /api/v1/auth/login` - 用户登录
- `POST /api/v1/auth/refresh` - 刷新令牌
- `POST /api/v1/auth/logout` - 用户登出
- `GET /api/v1/user/profile` - 获取用户信息
- `PUT /api/v1/user/profile` - 更新用户信息
- `GET /api/v1/user/preferences` - 获取用户偏好
- `PUT /api/v1/user/preferences` - 更新用户偏好

### 1.2 后端路由配置
**位置**: `src/internal/api/router.go`

后端实际路由：
- `POST /api/v1/users/register` ❌ **不匹配**
- `POST /api/v1/users/login` ❌ **不匹配**
- `POST /api/v1/users/refresh` ❌ **不匹配**
- `POST /api/v1/users/logout` ❌ **不匹配**
- `GET /api/v1/users/me` ❌ **不匹配**
- `PUT /api/v1/users/me` ❌ **不匹配**
- `GET /api/v1/users/preferences` ✅ **匹配**
- `PUT /api/v1/users/preferences` ✅ **匹配**

### 1.3 问题分析

#### 问题 1: 路由前缀不一致
- **前端期望**: `/api/v1/auth/*` 和 `/api/v1/user/*`
- **后端实际**: `/api/v1/users/*`
- **影响**: 所有认证相关的请求都会 404

#### 问题 2: 请求/响应字段不匹配

**注册接口**:
- 前端发送: `{ username, email, password, full_name }`
- 后端期望: `{ username, email, password }` (在 `request.UserRegister` 中)
- 后端响应: `{ access_token, refresh_token, token_type, expires_in, user }`
- 前端期望: 同上 ✅

**登录接口**:
- 前端发送: `{ username, password }` (但字段名为 username，实际可能是邮箱)
- 后端期望: `{ username, password }` (在 `request.UserLogin` 中)
- **问题**: 前端 Login.vue 使用 `username` 字段，但后端可能期望 `email` 或 `username`

#### 问题 3: 后端实现不完整
**位置**: `src/internal/api/v1/user.go`

- `Register` 方法: 只生成 token，**没有实际创建用户到数据库** ❌
- `Login` 方法: 使用硬编码的 `userID = "user_001"`，**没有验证用户凭证** ❌
- `GetCurrentUser` 方法: 返回硬编码的用户信息，**没有从数据库查询** ❌

### 1.4 数据库层问题

#### UserDAO 存在但未被使用
**位置**: `src/internal/repository/dao/user_dao.go`

UserDAO 已实现以下方法：
- `Create(user)` - 创建用户
- `FindByEmail(email)` - 根据邮箱查找
- `FindByID(id)` - 根据ID查找
- `Update(user)` - 更新用户
- `Delete(id)` - 删除用户
- `ExistsByEmail(email)` - 检查邮箱是否存在

**但是**: `src/internal/api/v1/user.go` 中的 `UserAPIEnhanced` 虽然有 `userDAO` 字段，但在 `Register` 和 `Login` 方法中**完全没有使用**！

#### 缺少 UserRepository 接口
- 存在 `ChatRepository` 和 `ResearchRepository` 接口
- **缺少** `UserRepository` 接口定义 ❌
- 只有 DAO 层，没有 Repository 层的抽象

#### 问题 1: 流式接口路径不一致

- **后端实际**: `POST /api/v1/chat/stream`
- **影响**: 流式聊天功能无法使用

#### 问题 2: ChatDAO 已实现但部分功能缺失

**位置**: `src/internal/repository/dao/chat_dao.go`

已实现的方法：
- ✅ `CreateSession` - 创建会话
- ✅ `GetSessionByID` - 获取会话
- ✅ `ListSessionsByUserID` - 获取用户会话列表

**缺失的方法**：
- ❌ `DeleteMessagesBySessionID` - 清空会话消息（前端有调用需求）

#### 问题 3: 数据模型不一致

**models.ChatSession** vs **model.ChatSession**:
- `src/internal/models/chat.go` - 被 Repository 使用
- `src/internal/repository/model/chat.go` - 被 DAO 使用

### 2.4 消息保存逻辑问题

**位置**: `src/internal/api/v1/chat.go`

- ⚠️ 而 `ChatDAO.CreateMessage` 不会自动增加计数

## 四、关键问题汇总

### 🔴 严重问题（阻塞功能）

1. **认证路由完全不匹配**
   - 前端: `/api/v1/auth/*` 和 `/api/v1/user/*`
   - 后端: `/api/v1/users/*`
   - 影响: 用户无法注册、登录

2. **注册和登录功能未实现**
   - `Register` 方法不创建用户到数据库
   - `Login` 方法不验证用户凭证
   - 影响: 认证系统完全不可用

3. **流式聊天路径不匹配**
   - 前端: `POST /api/v1/chat/chat/stream`
   - 后端: `POST /api/v1/chat/stream`
   - 影响: 流式聊天无法使用

### 🟡 中等问题（功能不完整）

4. **User 表字段缺失**
   - 缺少: `full_name`, `phone`, `role`, `status`, `is_admin`
   - 影响: 前端显示用户信息不完整

5. **UserDAO 未被使用**
   - UserDAO 已实现但 API 层不调用
   - 影响: 数据库操作无效

6. **缺少 UserRepository 接口**
   - 只有 DAO，没有 Repository 抽象层
   - 影响: 架构不一致

7. **ChatDAO 缺少清空消息方法**
   - 前端调用 `DELETE /sessions/:id/messages`
   - DAO 没有对应的 `DeleteMessagesBySessionID` 方法
   - 影响: 清空消息功能不可用

### 🟢 轻微问题（架构不一致）

8. **双重模型定义**
   - `models.ChatSession` vs `model.ChatSession`
   - 影响: 代码维护困难

9. **Repository 和 DAO 行为不一致**
   - `ChatRepository.SaveMessage` 自动增加计数
   - `ChatDAO.CreateMessage` 不增加计数
   - 影响: 可能导致数据不一致

## 五、修复建议

### 优先级 1: 修复认证系统

1. **统一路由前缀**
   ```go
   // 方案A: 修改后端路由（推荐）
   authGroup := v1Group.Group("/auth")
   {
       authGroup.POST("/register", r.authAPI.Register)
       authGroup.POST("/login", r.authAPI.Login)
       authGroup.POST("/refresh", r.authAPI.RefreshToken)
       authGroup.POST("/logout", r.authAPI.Logout)
   }
   
   userGroup := v1Group.Group("/user")
   userGroup.Use(middleware.Auth())
   {
       userGroup.GET("/profile", r.authAPI.GetCurrentUser)
       userGroup.PUT("/profile", r.authAPI.UpdateProfile)
       // ...
   }
   ```

2. **实现真实的注册逻辑**
   ```go
   func (api *UserAPIEnhanced) Register(c *gin.Context) {
       var req request.UserRegister
       // ... 验证请求
       
       // 检查邮箱是否已存在
       exists, _ := api.userDAO.ExistsByEmail(ctx, req.Email)
       if exists {
           pkg.BadRequest(c, "邮箱已被注册")
           return
       }
       
       // 哈希密码
       hashedPassword, _ := bcrypt.GenerateFromPassword([]byte(req.Password), bcrypt.DefaultCost)
       
       // 创建用户
       user := &model.User{
           ID:       pkg.GenerateUUID(),
           Email:    req.Email,
           Username: req.Username,
           Password: string(hashedPassword),
       }
       
       if err := api.userDAO.Create(ctx, user); err != nil {
           pkg.InternalError(c, "创建用户失败")
           return
       }
       
       // 生成 token
       accessToken, _ := api.jwtManager.GenerateToken(user.ID, user.Email, user.Username)
       // ...
   }
   ```

3. **实现真实的登录逻辑**
   ```go
   func (api *UserAPIEnhanced) Login(c *gin.Context) {
       var req request.UserLogin
       // ... 验证请求
       
       // 查找用户（支持用户名或邮箱登录）
       user, err := api.userDAO.FindByEmail(ctx, req.Username)
       if err != nil {
           // 尝试用户名查找
           user, err = api.userDAO.FindByUsername(ctx, req.Username)
           if err != nil {
               pkg.Unauthorized(c, "用户名或密码错误")
               return
           }
       }
       
       // 验证密码
       if err := bcrypt.CompareHashAndPassword([]byte(user.Password), []byte(req.Password)); err != nil {
           pkg.Unauthorized(c, "用户名或密码错误")
           return
       }
       
       // 生成 token
       accessToken, _ := api.jwtManager.GenerateToken(user.ID, user.Email, user.Username)
       // ...
   }
   ```
