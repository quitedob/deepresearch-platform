## 系统架构

### 1. 核心组件

**ResearchService** (`src/internal/service/research_service.go`)
- 管理研究会话生命周期
- 协调 Agent 执行
- 处理进度回调和事件推送

**ResearchAgent** (`src/internal/eino/agent/research.go`)
- 基于 Eino ReAct Agent
- 执行多轮推理和工具调用
- 生成结构化研究报告

**EventStream** (`src/internal/service/stream_manager.go`)
- SSE 事件流管理
- 实时推送研究进度

### 2. 研究工具

**WebSearch** (`src/internal/eino/tool/websearch.go`)
- 使用智谱AI的 web_search 功能
- 获取最新网络信息

**ArXiv** (`src/internal/eino/tool/arxiv.go`)
- 搜索学术论文
- 提供理论深度

**Wikipedia** (`src/internal/eino/tool/wikipedia.go`)
- 查询百科知识
- 提供基础定义

### 3. API 端点

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/v1/research/start` | POST | 启动研究任务 |
| `/api/v1/research/status/:id` | GET | 获取研究状态 |
| `/api/v1/research/stream/:id` | GET | SSE 流式进度 |
| `/api/v1/research/sessions` | GET | 获取会话列表 |
| `/api/v1/research/export/:id` | GET | 导出研究结果 |

## 配置要求

### config.yaml

```yaml
llm:
  providers:
    zhipu:
      api_key: "your_zhipu_api_key"  # 必需
      base_url: "https://open.bigmodel.cn/api/paas/v4"
      models:
        - glm-4.6        # 研究用主模型
        - glm-4.5-air    # 备用模型

research:
  max_iterations: 10      # Agent 最大迭代次数
  session_timeout: 3600   # 会话超时（秒）
```

## 使用示例

### 前端调用

```javascript
// 启动研究
const response = await researchAPI.startResearch({
  query: "What is artificial intelligence?",
  research_type: "deep",
  llm_config: {
    provider: "zhipu",
    model_name: "glm-4.6"
  }
});

const sessionId = response.session_id;

// 监听进度（SSE）
const eventSource = new EventSource(
  `/api/v1/research/stream/${sessionId}?token=${token}`
);

eventSource.onmessage = (event) => {
  const data = JSON.parse(event.data);
  if (data.type === 'completed') {
    console.log('研究完成:', data.data.report_text);
  }
};
```

## 研究流程

1. **用户提交查询** → API 接收请求
2. **创建研究会话** → 保存到数据库
3. **启动 Agent** → 异步执行研究
4. **ReAct 循环**：
   - Agent 分析问题
   - 决定使用哪个工具
   - 调用工具获取信息
   - 综合信息继续推理
   - 重复直到得出结论
5. **生成报告** → 结构化输出
6. **保存结果** → 数据库持久化
7. **推送完成事件** → SSE 通知前端
