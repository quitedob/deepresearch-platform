# 聊天记忆功能实现总结 (更新版)

## 功能概述

实现了完整的聊天记忆功能，包括：

1. **记忆功能开关** - 用户可在设置中启用/禁用聊天记忆
2. **上下文累加** - 自动将历史聊天记录塞入上下文
3. **128K限制处理** - 超限时提醒用户，支持总结后新建对话
4. **自定义提示词** - 用户级别的自定义系统提示词
5. **System提示词层级** - 严格的优先级顺序

## 提示词优先级

发送给LLM的消息顺序：
1. `.env` 中的 `SYSTEM_SAFETY_PROMPT`（最高优先级，安全提示词）
2. 用户自定义提示词（`custom_system_prompt`）
3. 会话系统提示词（`session.system_prompt`）
4. 历史聊天记录（如果启用记忆功能）
5. 当前用户消息

## 使用流程

1. 用户在设置中启用/禁用记忆功能
2. 可设置自定义提示词（每次对话自动添加）
3. 聊天时自动累加历史记录到上下文
4. 接近128K限制时显示警告
5. 超过限制时弹窗提醒，用户确认后：
   - 自动总结当前对话
   - 创建新会话（携带总结）
   - 切换到新会话继续对话
