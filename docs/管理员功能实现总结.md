# 管理员功能实现总结

## 功能概述

本次实现了完整的管理员后台和会员体系功能，包括：

### 1. 会员体系
- **普通用户**: 默认10次聊天（普通+深度思考合计）+ 1次深度研究
- **高级会员**: 默认5小时内50次聊天 + 10次深度研究，支持付款和激活码两种激活方式

### 2. 配额配置功能（新增）
管理员可以灵活配置配额：
- **全局配置**：设置每个会员层级的默认配额（聊天次数、研究次数、重置周期）
- **应用到所有用户**：将全局配置一键应用到所有该类型的用户
- **个人配额**：针对具体用户单独设置配额（覆盖全局配置）
- **批量设置**：批量为多个用户设置配额

### 3. 管理员功能
- 模型配置：控制ollama、deepseek、zhipu三个提供商的模型对用户可见性
- 用户管理：查看所有用户、修改状态（禁用/激活）、修改会员状态
- 聊天记录：查询和导出用户聊天记录JSON
- 激活码管理：创建激活码、设置激活人数、追踪激活用户
- 配额管理：编辑和重置用户配额（支持全局和个人配置）
- 通知系统：一键发送通知给所有用户

## API端点

### 管理员API (`/api/v1/admin`)
- `GET /stats` - 获取统计信息
- `GET /users` - 获取用户列表
- `PUT /users/:id/status` - 更新用户状态
- `PUT /users/:id/membership` - 更新会员状态
- `POST /users/:id/reset-quota` - 重置配额
- `PUT /users/:id/quota` - 设置配额
- `GET /users/:id/chat-history` - 获取聊天记录
- `GET /users/:id/chat-history/export` - 导出聊天记录
- `GET /activation-codes` - 激活码列表
- `POST /activation-codes` - 创建激活码
- `GET /activation-codes/:id` - 激活码详情
- `PUT /activation-codes/:id` - 更新激活码
- `DELETE /activation-codes/:id` - 删除激活码
- `GET /notifications` - 通知列表
- `POST /notifications` - 创建通知
- `DELETE /notifications/:id` - 删除通知
- `GET /providers` - 提供商配置
- `PUT /providers` - 更新提供商配置
- `GET /models` - 模型配置
- `PUT /models` - 更新模型配置
- `GET /quota-configs` - 获取配额配置
- `PUT /quota-configs` - 更新配额配置（支持应用到所有用户）
- `PUT /users/:id/custom-quota` - 设置用户自定义配额
- `PUT /users/batch-quota` - 批量设置用户配额

### 用户端会员API (`/api/v1/membership`)
- `GET /` - 获取会员信息
- `GET /quota` - 获取配额信息
- `POST /activate` - 使用激活码
- `GET /check-chat-quota` - 检查聊天配额
- `GET /check-research-quota` - 检查研究配额

### 用户端通知API (`/api/v1/notifications`)
- `GET /` - 获取通知列表
- `GET /unread-count` - 获取未读数量
- `POST /:id/read` - 标记已读
- `POST /read-all` - 全部标记已读

## 使用说明

1. 重启后端服务，数据库会自动迁移新表
2. 在数据库中将某个用户的 `is_admin` 字段设为 `true` 使其成为管理员
3. 管理员登录后可在侧边栏看到⚙️图标，点击进入管理控制台
4. 用户可在侧边栏看到通知铃铛和会员信息
