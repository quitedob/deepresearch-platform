# 问题修复清单

## 修复完成时间
2024-12-02

## 后端修复

### 1. ActivationCodeDAO.UseCode - 添加重试机制
**文件**: `src/internal/repository/dao/activation_code_dao.go`

**问题**: 虽然使用了FOR UPDATE，但缺少重试机制，高并发下容易导致deadlock，用户激活失败后没有合理的重试指示。

**修复内容**:
- 添加了详细的错误类型定义（ErrCodeNotFound, ErrCodeInactive, ErrCodeExpired等）
- 实现了 `UseCodeWithRetry` 方法，支持最多3次重试
- 使用指数退避策略（50ms, 100ms, 200ms）
- 返回 `UseCodeResult` 结构，包含重试次数和详细错误码
- 使用 `FOR UPDATE NOWAIT` 避免长时间等待锁

### 2. ChatDAO - 强化参数验证
**文件**: `src/internal/repository/dao/chat_dao.go`

**问题**: ListSessionsByUserID中使用了ORDER BY直接传参，缺少完整的参数验证。

**修复内容**:
- 添加排序字段白名单 `allowedSortFields`
- 添加排序方向白名单 `allowedSortOrders`
- 实现 `ValidateSortParams` 函数验证排序参数
- 新增 `ListSessionsByUserIDWithSort` 方法支持安全的自定义排序

### 3. ChatService - LRU缓存机制
**文件**: `src/internal/service/chat_service.go`

**问题**: 所有消息历史都缓存，导致内存占用过高，缺少LRU清理机制。

**修复内容**:
- 实现了 `LRUMessageCache` 结构，支持容量限制和大小限制
- 默认最多缓存100个会话，最大50MB
- 自动淘汰最久未使用的缓存条目
- 添加 `GetMessageCacheStats` 方法获取缓存统计
- 添加 `InvalidateSessionCache` 方法主动清除缓存

### 4. validation.go - 完善ID验证
**文件**: `src/internal/pkg/utils/validation.go`

**问题**: ValidateAndSanitizeID验证可能不完整。

**修复内容**:
- 增强验证逻辑，检查空字符
- 添加 `IsValidNumericID` 支持纯数字ID
- 添加 `ValidateAndSanitizeIDs` 批量验证方法

## 前端修复

### 5. Home.vue - 会话校验和请求防抖
**文件**: `vue/src/views/Home.vue`

**问题**: 
- handleEditAndSend 和 handleRegenerate 函数中的 SessionId 校验逻辑不完整
- 快速点击操作时可能发送重复请求
- 网络错误后没有自动重试或保存草稿

**修复内容**:
- 添加请求防抖机制（500ms）
- 实现草稿自动保存和恢复
- 添加会话切换时的请求中止逻辑
- 实现网络错误自动重试（最多3次，指数退避）
- 为消息添加 `sessionId` 属性用于校验
- 监听会话切换事件，自动清理进行中的请求
